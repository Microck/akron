title: Manage .akr setups

Akron setup state covers the options, bindings, StartPos slots, and scoped tool settings that can be exported into `.akr` packs. Use setup backups to separate clean-play guardrails, room-lab tools, accessibility settings, and map-making workflows.

## Start From A Setup

Use `.akr` setup backups to separate workflows. The Akron overlay shows the current policy/status state and supports `.akr` import/export, but player setup should be organized around visible tool groups and scoped packs. Rulesets are advanced policy presets behind some setups, not the main setup-management surface.

| Setup | Best for |
|---|---|
| Default setup | Learning Akron and ordinary play. |
| Room-lab setup | StartPos, reloads, timescale, and route iteration. |
| Clean-run setup | Attempts where clean classification matters. |
| Local testing setup | Experiments and unrestricted tool testing. |
| Map inspection setup | Capture, inspection, and map development workflows. |
| Accessibility-focused setup | Visual, audio, and comfort settings. |

## Maintain Setups

<Steps>
  <Step title="Start from the closest setup">
    Use an existing setup or export a fresh baseline before changing individual tools for a new workflow.
  </Step>
  <Step title="Change one tool group at a time">
    Configure overlay behavior, HUD tools, room-lab tools, visual settings, and interop separately so you can find mistakes.
  </Step>
  <Step title="Export a backup">
    Export a whole `.akr` setup backup when the setup is stable.
  </Step>
  <Step title="Use scoped imports">
    Import only the section you need when applying shared StartPos, keybind, Auto Kill, Auto Deafen, recorder, audio, or HUD packs.
  </Step>
</Steps>

## Reset Strategy

Use reset actions when setup state has drifted too far from the intended use. Prefer exporting a backup first if it contains map-specific StartPos slots or carefully tuned binds.

## Clean Setup Hygiene

- Keep state-changing tools out of `Leaderboard-clean`.
- Check suboptions, not only parent rows.
- Avoid importing whole setup packs from untrusted sources into a clean-run setup.
- Use [Feature status reference](/reference/feature-status-reference) when deciding where a setting belongs.
