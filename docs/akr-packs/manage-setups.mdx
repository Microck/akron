---
title: Manage .akr setups
description: Back up, reset, and maintain Akron setup state for different play contexts.
---

Akron setup state covers the options, bindings, StartPos slots, and scoped tool settings that you can export into `.akr` packs. Use setup backups to separate clean-play guardrails, room-lab tools, accessibility settings, and map-making workflows.

## Start from a profile

Use Akron profiles and `.akr` setup backups to separate workflows. The Akron overlay shows the current profile and policy/status state, and supports `.akr` import/export. Organize your setups around visible tool groups and scoped packs. Rulesets are advanced policy presets behind some profiles, not the main setup-management surface.

| Profile | Best for |
|---|---|
| Casual | Learning Akron and ordinary play. |
| Practice | StartPos, reloads, timescale, and route iteration. |
| Leaderboard Clean | Attempts where clean classification matters. |
| Sandbox | Experiments and unrestricted tool testing. |
| Map Maker | Capture, inspection, and map development workflows. |
| Accessibility | Visual, audio, and comfort settings. |

## Maintain setups

<Steps>
  <Step title="Start from the closest profile">
    Use an existing profile or export a fresh baseline before changing individual tools for a new workflow.
  </Step>
  <Step title="Change one tool group at a time">
    Configure overlay behavior, HUD tools, room-lab tools, visual settings, and interop separately so you can find mistakes.
  </Step>
  <Step title="Export a backup">
    Export a whole `.akr` setup backup when the setup is stable.
  </Step>
  <Step title="Use scoped imports">
    Import only the section you need when applying shared StartPos, Keybinds, Auto Kill, Auto Deafen, Recorder, Audio, or HUD packs.
  </Step>
</Steps>

## Reset strategy

Use reset actions when setup state has drifted too far from the intended use. Prefer exporting a backup first if it contains map-specific StartPos slots or carefully tuned binds.

## Clean setup hygiene

- Keep state-changing tools out of Leaderboard Clean.
- Check suboptions, not only parent rows.
- Avoid importing whole setup packs from untrusted sources into a Leaderboard Clean setup.
- Use the [Feature status reference](/reference/feature-status-reference) when deciding where a setting belongs.
