Akron capture tools support review, sharing, proof context, and creator workflows. Capture output does not change gameplay unless a specific suboption states otherwise.

## Capture Surfaces

| Surface | Use it for |
|---|---|
| Screenshot tool | Capture the current game view for review or sharing. |
| Room capture | Capture a single room view or room marker output. |
| Map capture | Capture chapter-wide map output through the screenshot scanner. |
| Internal recorder | Record local frames with audio, replay buffer, codec selection, and event-based clip triggers. |
| Proof sidecar | Export a JSON file with classification, active-feature list, and proof telemetry. |

## Capture Workflow

<Steps>
  <Step title="Open the capture surface">
    Use the overlay row or a configured shortcut for the capture tool you need.
  </Step>
  <Step title="Configure output">
    For screenshots, room capture, and map capture, set the output folder, filename template, scale, and preview visibility. For recording, configure codec, replay buffer duration, and clip triggers.
  </Step>
  <Step title="Check policy suboptions">
    Timer-freeze or proof-affecting suboptions can be stricter than the base capture row. Read the badge before enabling.
  </Step>
  <Step title="Capture and review">
    Inspect the output before sharing. Confirm it shows the intended state and does not include unwanted technical details.
  </Step>
</Steps>

## Internal Recorder

The internal recorder captures local frames for review without changing gameplay state. It provides output, audio, replay buffer, codec, clip trigger, and preset configuration through the **Internal Recorder** overlay tab.

Preset shortcuts (**CPU**, **NVIDIA**, **AMD**) select a codec and quality combination in one step. Available codecs include x264 (CPU), H.264 NVENC (NVIDIA), H.264 AMF (AMD), HEVC NVENC, and VP9.

For proof-oriented use, combine recorder settings with **Submission Mode** and proof helpers. A video alone does not capture all proof context.

## Streamer Mode and Output Paths

When **Streamer Mode** is active, Akron replaces local filesystem paths with filenames in HUD, overlay, and proof displays. Streamer Mode does not hide in-game data such as room IDs, flags, entity text, or StartPos coordinates.

## Troubleshooting

- If a capture is blank or cropped, verify the capture mode and output resolution.
- If output paths appear hidden, check whether **Streamer Mode** is active.
- If proof capture warns that recording is not armed, enable **Proof Recorder Guard** or review the recorder settings.
- If capture freezes timers, check whether a stricter suboption is enabled.
