Akron gameplay tools help with routing, learning, and map inspection. Using many of these tools marks the active attempt as **Cheat** because they change state, timing, position, or execution.

## Common Tools

| Tool | Use it for | Policy |
|---|---|---|
| Retry hotkey | End the current attempt through Celeste's retry flow. | Normal clear when it matches the configured allowed behavior. |
| Room reload | Restart the current room outside normal play. | Cheat. |
| Chapter reload | Restart the current chapter through Celeste's loader. | Normal clear. |
| Room warp | Teleport to a selected room. | Cheat. |
| Click Teleport, Cursor Zoom, and Entity Inspector | Use the cursor for quick position, camera inspection, and object inspection tools. Cursor Tools can temporarily enable selected cursor-driven tools while held, with one left-click action selected at a time. | Cheat. |
| Free Camera | Move the room camera independently with movement input or optional mouse control. | Cheat. |
| Hitbox viewer | Draw collision and hazard hitboxes for timing and spatial review. | Cheat. |
| Entity Inspector | Hold the inspector cursor bind to pin entities or triggers, highlight hovered and pinned targets, and inspect runtime and map-source details. | Cheat. |
| StartPos tools | Capture and restore StartPos snapshots, including smart same-room respawn selection. | Cheat. |
| Frame advance and freeze | Freeze gameplay updates and advance one frame at a time. | Cheat. |
| Timescale | Change gameplay update speed between 0.1× and 2.0×. | Cheat. |
| Auto Kill | Kill the player automatically after a configured map-time threshold or inside selected regions. | Cheat. |
| Auto Deafen | Press the configured Discord deafen hotkey when entering a selected region. Stays active until death or reset. | Normal clear. |

## Room-Lab Workflow

<Steps>
  <Step title="Use a non-submission setup">
    Use a setup intended for state-changing tools before enabling room-lab features. This keeps the attempt indicator accurate.
  </Step>
  <Step title="Choose the right tool for the job">
    Use **StartPos** for repeated room or state-specific routing. Use **Room reload** for quick reset loops. Use **Chapter reload** when a full chapter restart matches the intended clean workflow.
  </Step>
  <Step title="Watch the status indicator">
    State-changing tools - StartPos restore, Room reload, frame tools, and timescale - mark the active attempt as Cheat. Check the status label before resuming a clean attempt.
  </Step>
  <Step title="Keep exports scoped">
    Export only the relevant StartPos snapshot or section when sharing map-specific setups.
  </Step>
</Steps>

## StartPos Safety

StartPos is Akron's built-in workflow for saving and restoring room positions. Each StartPos snapshot stores the full room state - not just coordinates - so repeated attempts start from the intended setup.

If StartPos restore is blocked, check the visible policy status, the map compatibility settings, and whether the saved snapshot belongs to the current map and room.

See [StartPos recovery](/troubleshooting/startpos-recovery) for recovery guidance.

## Input Shortcut Caution

Shortcuts that synthesize or modify execution inputs are Cheat. This includes input-assist shortcuts such as Neutral Drop and Backboost.

Check [Clean vs cheat classifications](/concepts/clean-vs-cheat) before binding a shortcut for submitted-run use.
