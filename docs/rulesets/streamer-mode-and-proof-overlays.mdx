title: Streamer Mode, Proof Mode, and Low-Distraction Overlays
description: Reference for Streamer Mode, Proof Mode, and Low-Distraction overlay behaviors in Akron.
---

Overlay rulesets change presentation and proof output without replacing the primary ruleset.

## Streamer Mode

`Streamer Mode` hides local filesystem paths in Akron-owned UI and proof output. When active, Akron displays only the filename or final folder name instead of the full path.

Use this for streams, videos, screenshots, or public proof packages where local capture or export paths should not be exposed.

`Streamer Mode` is not a general privacy mode. It does not inspect other mods, external overlays, visible game room data, or arbitrary text on screen.

## Proof Mode

`Proof Mode` keeps proof surfaces compact and ruleset-aware. Akron normally enables this indirectly through `Submission Mode` or imported community rulesets. The direct command path exists for QA and review automation rather than as a primary player setting.

Proof Mode does not:

- Certify legitimacy.
- Spoof completion.
- Change save stats.
- Replace moderator review.

## Low-Distraction State

The Low-Distraction State activates when all five visual-noise channels are suppressed (particles, trails, glitch, anxiety, and distortion). It is a derived presentation state, not a selectable ruleset or separate proof authority.

## When to Use These Overlays

| State | Use Case |
|---|---|
| Streamer Mode | Share screenshots, video, streams, or proof artifacts while hiding local filesystem paths. |
| Proof Mode | Use when Submission Mode or community rulesets require compact proof metadata. |
| Low-Distraction | Use once visual-noise settings form a lower-noise setup across play sessions. |
