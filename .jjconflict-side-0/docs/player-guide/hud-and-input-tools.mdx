Akron HUD tools display local state and review information. Each surface has a clear policy: **Goldberry/Hardlist clear**, **Normal clear**, or **Cheat**. Cheat surfaces expose precise resources or hidden information.

## HUD Surfaces

| Surface | Use it for | Policy |
|---|---|---|
| Room labels | Show passive room information. | Normal clear. |
| Input viewer | Show current local inputs as a text chord. | Goldberry/Hardlist clear. |
| Control display | Show a visual overlay of pressed keys or buttons. | Goldberry/Hardlist clear. |
| Input history | Review recent input state in a scrolling panel. | Goldberry/Hardlist clear for the base feature. |
| Inputs per second | Monitor your current input press rate. | Normal clear. |
| Room timer | Show timing information for the current room. | Normal clear. |
| Room stat tracker | Show room name, deaths, timer, berry count, and alive time. | Normal clear. |
| Death stats | Show death counters. | Goldberry/Hardlist clear for the base feature. |
| Attempts | Show the current attempt number. | Goldberry/Hardlist clear. |
| Status labels | Show profile, ruleset, and attempt status. | Normal clear. |
| Custom HUD labels | Show configured local status labels with template variables. | Normal clear. |
| Stamina, dash, speed resources | Show exact resource state. | Cheat. |

## Configure HUD Output

<Steps>
  <Step title="Check the visible status">
    Check the attempt status and policy state before enabling submission-oriented displays or attempt-changing widgets.
  </Step>
  <Step title="Enable one surface at a time">
    Turn on the base row in the Labels tab, then open the popup to configure rows, labels, colors, or display limits.
  </Step>
  <Step title="Check suboptions">
    Read any suboption policy badges. History pinning, timer freeze, and similar behavior can carry a different policy from the parent.
  </Step>
  <Step title="Test in a room">
    Close the overlay and confirm the widget appears where expected without covering gameplay-critical information.
  </Step>
</Steps>

## Input Tools

Use the **input viewer** to see your current inputs as a text chord. Use **input history** to review recent inputs after movement, death, or a routed attempt. Enable the **control display** for a visual overlay of which keys or buttons are pressed. Check **inputs per second** to monitor your press rate.

Input displays show local inputs without changing gameplay. Anything that synthesizes or redirects inputs sets the attempt status to Cheat.

## Room Stats and Labels

The **Room Stat Tracker** shows room name, deaths, timer, berry count, and alive time in the label HUD. Its freeze behavior is a tracker suboption with its own policy badge - check it separately from the base label.

Use [Custom labels](/feature-guide/custom-labels) when you want to build your own HUD text from Akron variables, event triggers, and label-specific styling.

## Streamer Mode

`Streamer Mode` hides local filesystem paths in Akron-owned UI and proof output. It is a presentation setting, not a gameplay mutation or a general privacy mode.
